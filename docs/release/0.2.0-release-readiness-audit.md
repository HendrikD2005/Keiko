## Executive Decision

The previously open release findings are addressed in this remediation branch:

- #802 proxy- and custom-CA-aware outbound HTTP egress is implemented in the model gateway transport and wired into model calls, embeddings, gateway setup discovery, and Figma snapshot fetch/render ports, including Figma-only startup where no model provider is configured.
- #121 mobile root toolbar clipping is addressed with a narrow-viewport layout compression rule and a CSS regression test.
- The release workflow is now explicit, gated, and beta-safe: prerelease versions are blocked from publishing with the `latest` npm dist-tag, and the default manual publish tag is `beta`.

This branch should not be treated as an automatic production `GO`. The final release gate must still rerun the full CI/release workflow on the merge target, plus a real enterprise Figma validation using the target proxy/custom-CA environment. Until those final checks complete, the correct release posture is `HOLD`, not `GO`.

For the planned community beta, use lowercase semver `0.2.0-beta.0` and publish under the npm `beta` dist-tag. Do not publish a prerelease on `latest`.

## Security And Privacy Review

- `.env`, `.keiko/`, and `/only-for-internal-use/` remain ignored by Git and were not copied into the repository.
- Proxy URLs and CA bundle paths are redacted from setup responses and config summaries.
- Proxy credentials embedded in proxy URLs are rejected rather than logged or persisted.
- Outbound egress failures are coded and bounded; raw tokens, headers, proxy credentials, and certificate contents are not exposed in thrown errors.
- Figma board IDs, screenshots, local test data, tokens, and customer data are not written into this report or committed artifacts.
- Quality Center external writes remain dry-run gated unless explicitly enabled by runtime configuration.

## Final Memo

Known remediation findings are implemented and ready for PR review. The release should stay on `HOLD` until the merged release candidate passes the full release workflow and a live enterprise Figma proxy/custom-CA smoke test.

Recommended beta path after merge and approval:

1. Merge this PR to `dev`.
2. Promote `dev` to `main` using the repository's normal protected-branch process.
3. Set the package version to `0.2.0-beta.0`.
4. Create tag `v0.2.0-beta.0`.
5. Run the release workflow manually on that tag with `publish=true` and `npm_dist_tag=beta`.
