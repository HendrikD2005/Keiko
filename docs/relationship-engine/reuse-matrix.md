## Legend

- `adopt-unchanged` â€” use the existing surface as-is. No contract or implementation change.
- `extend-with-contract` â€” add an additive type or pure helper in an existing package (no breaking change).
- `generalize-port` â€” introduce a port the existing package exposes so the relationship engine can compose without taking a direct dependency on the implementation.
- `new-capability-gap` â€” no existing surface fits. The gap is enumerated in [gap-analysis.md](gap-analysis.md) with the smallest possible new surface.

## How this matrix constrains the downstream issues

- #534 has no degree of freedom on the closed-set pattern, the runtime-iterable-array pattern, or the validator-result pattern â€” they are matrix rows 2, 3, 28.
- #535 has no degree of freedom on the storage substrate (`node:sqlite` + `--experimental-sqlite` + `PRAGMA user_version`), the schema convention (STRICT + FK + dual index), or the policy decision shape â€” they are matrix rows 4, 6, 23, 24. The only open choice is the endpoint-resolver port shape (row 5) and the ADR that records it.
- #536 has no degree of freedom on the redactor (`createAuditRedactor` + `deepRedactStrings`) or the body-free invariant, and row 9 records the optional evidence-manifest seam as additive rather than shipped current behavior.
- #537 / #540 have no degree of freedom on the UI substrate (ADR-0026 locks it) or the descriptor invariants (ADR-0029 locks them). The Wave 1 assumption about two new `WindowType` entries was superseded by the current panel-based integration on `dev`.
- #541 has no degree of freedom on event-name discipline, redaction, or the body-free invariant.
- #542 has no degree of freedom on bounded traversal, terminal-policy reuse, or model-gateway isolation.
- #543 has no degree of freedom on the existing test gates (`npm test`, `npm run lint`, `npm run typecheck`, `npm run arch:check`, `npm run arch:check:negative`, `npm run build`, `npm pack`).

Current-state note: later implementation narrowed some open options above. Storage is now governed by [ADR-0031](../adr/ADR-0031-relationship-storage-and-validation.md), audit/activity by [ADR-0032](../adr/ADR-0032-relationship-audit-and-activity-model.md), and UI containment by [ADR-0033](../adr/ADR-0033-relationship-ui-containment.md).
