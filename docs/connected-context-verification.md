# Connected repository context verification matrix (#177)

## Context

Epic [#177](https://github.com/oscharko-dev/Keiko/issues/177) defined the connected-repository-context substrate that Conversation Center uses to ground assistant answers in real code from the user's workspace. Children [#178](https://github.com/oscharko-dev/Keiko/issues/178)–[#187](https://github.com/oscharko-dev/Keiko/issues/187) shipped the contracts, search facade, structural adapters, planner/governor, candidate ranker, micro-index/assembler, Files-window connector UI, grounded Q&A BFF route, governed workflow handoff contracts, and privacy/audit controls. This matrix is the closure deliverable for [#188](https://github.com/oscharko-dev/Keiko/issues/188): it cross-references every epic-level acceptance criterion to file:line evidence in the merged code, captures the cold-cache CI-parity gauntlet, and records the six regression fixtures named in #188's deliverables.

The format mirrors [`docs/architecture-sprint-verification.md`](architecture-sprint-verification.md), which closed Epic #156 the same way.

- Grounded answer presentation: [`packages/keiko-ui/src/app/components/desktop/GroundedAnswer.test.tsx`](../packages/keiko-ui/src/app/components/desktop/GroundedAnswer.test.tsx) plus the jest-axe smoke at [`packages/keiko-ui/src/app/components/desktop/GroundedAnswer.a11y.test.tsx`](../packages/keiko-ui/src/app/components/desktop/GroundedAnswer.a11y.test.tsx)

A live browser-smoke session was not automated in this PR. The manual smoke procedure is:

1. `keiko ui` (binds the default loopback port).
2. Open `/`, create a project, create a chat.
3. Open the Files window; select two paths inside the project; click `Connect`.
4. Type a natural-language question and submit.
5. Confirm the grounded answer renders with at least one citation, the context inspection summary card, and no error states.

Step 5 exercises the full pipeline shipped by #178–#187; failure at any step is a regression in one of those PRs, not a new finding for #188.

## Security review

The security posture of the connected-context surface is summarised in three layers:

- **Wire surface** (`GroundedAnswerContextPackSummary`): structurally counts-only by construction. Verified by the structural redaction tests in [`packages/keiko-contracts/src/bff-wire.test.ts`](../packages/keiko-contracts/src/bff-wire.test.ts) which run an attacker-controlled pack with `scopeId`, `workspaceRoot`, and `query.text` set to secret-shaped values and assert none of those fields reaches the summary.
- **BFF boundary**: `deps.redactor` (the `redact` + `deepRedactStrings` composition from `@oscharko-dev/keiko-security`) is now applied to `uncertainty.claim` at [`packages/keiko-server/src/grounded-qa.ts:191`](../packages/keiko-server/src/grounded-qa.ts). The five attacker-controlled tests at [`packages/keiko-server/src/grounded-qa.redaction.test.ts:208`](../packages/keiko-server/src/grounded-qa.redaction.test.ts) feed every contract-accepted string field with a secret shape from the keiko-security taxonomy (`sk-…`, `ghp_…`, AKIA, xoxb, AIza, Bearer, PEM) and assert the wire response is clean.
- **Evidence ledger**: grounded answers write a `connectedContext` manifest section with redacted paths, query hashes, excerpt hashes, tools, and counts. [`packages/keiko-evidence/src/connected-context-evidence.test.ts`](../packages/keiko-evidence/src/connected-context-evidence.test.ts) asserts query text and excerpt content are not persisted.
- **Architecture**: ADR-0019 direction-1 (contracts as a leaf) is intact across the epic. `arch:check` reports zero violations across 584 modules and 1311 dependencies; the negative gate fires 12/12 expected fixtures.

ADR-0022 records the decisions and the structural redaction guarantee (D1 no full pack
ledger, D2 wire-only summary, D3 one-way evidence reference, D3.1 process-local
micro-index cleanup, D3.2 metadata-only evidence, D4 BFF boundary redaction).

## Limitations and follow-ups

- **Persistent pack ledger** — Full `ConnectedContextPack` objects are not persisted by ADR-0022 D1. If an enterprise audit policy ever requires post-hoc pack inspection, that work belongs in a new issue (not in #177's scope).
- **Evidence-run cleanup on chat delete** — Evidence runs survive chat deletion intentionally (ADR-0022 D3). [#154](https://github.com/oscharko-dev/Keiko/issues/154) is the home for the "clear vs delete chat" distinction; the one-way reference is the documented design until that issue ships.
- **Subprocess-backed ripgrep adapter** — The search facade today uses an in-process scanner. A subprocess-backed adapter for very large repositories is deferred and tracked on the #179 closure thread.
- **Query text in wire shape** — `GroundedAnswerContextPackSummary` carries `queryKind` only. The evidence ledger stores query text hash/byte count for audit correlation, not the raw query text.
- **Live browser smoke automation** — The component-test layer covers presentational and behavioural state transitions; a true browser-driver smoke (Playwright/Chrome MCP) for Files-window → grounded-Q&A is deferred to [#142](https://github.com/oscharko-dev/Keiko/issues/142) Conversation Center integration, which is the downstream consumer of this substrate.
- **End-to-end workflow handoff execution** — #186 shipped the contracts; the BFF route that accepts a handoff request and the workflow-side enforcement of `PatchScope` violations are still to be wired (downstream consumer issues, not blockers on #177).
